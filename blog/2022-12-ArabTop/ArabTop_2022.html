Textual Visual Semantic Relatedness Dataset for Image Captioning

In this blog post, I will describe a simple out-of-the-box construction of a textual visual context dataset for image captioning. This dataset is based on the COCO , and we (our work) further extend the dataset using state-of-the-art and out-of-the-box tools to extract the most related visual semantic context information from each image. Note that, this dataset is an extension of a dataset that is built for a previous approach to be fine-tuned, but It can be used for any task.

Our code is available on Github and for quick start try Colab .

This combined visual context dataset provides the language and vision research community the chance to use semantic similarity at text level between text and image to improve the results. Also, unlike the computer vision community that tackles this problem by relying on visual features , our approach relies on textual information and uses existing state-of-the-art visual context methods. Therefore, it can be used as end-to-end or post-processing. In addition, we propose a similarity based re-ranking task, where the main concept is to learn a scoring function that assigns higher similarity to better caption hypotheses that correlate with its visual context from the same image.

Visual Context Information#

To obtain the visual context from each image, we will use out-of-the-box classifiers to extract the image context information. We extract two kinds of context information: object and scenarios present/seen in the image.

ResNet-152 . A residual or shortcut connection-based deep network that relies heavily on batch normalization. The shortcut is known as a gated recurrent unit that allows training deeper network while maintaining less complexity than VGGNet.

CLIP. (Contrastive Language-Image Pretraining) It is a pre-trained model with contrastive loss where the pair of image-text needs to distinguish from randomly selected sample pairs. CLIP uses available resources across the Internet without human annotation 400M pairs. CLIP achieves state-of-the-art in a wide range of image classification tasks in a zero-short way.

Inception-Resnet FRCNN . An improved variant of Faster R-CNN, with a trade-off of abetter accuracy and fast inference, via high-level feature extracted from Inception-Resnet. Faster R-CNN has two stages (1) a region proposal that suggests region, and (2) region-of-interest scoring. It is a pre-trained model that trained on COCO categories with 80 object categories.

The objects extracted from all pre-trained, mentioned above, are obtained by extracting top-3 object class/category from each classifier after filtering out no confident instance via probability threshold < 0.2. Note that, all the textual visual context extracted pre-trained models, mentioned above, are fast at inference time, which is suitable for new task adoption. Therefore, we evade a computational hungry model, such as BUTD (Bottom-Up Top-down Feature) as it is computational expansive for our task.

Resulting Dataset

COCO-visual. It consists of 413915 captions with associated visual context top-3 objects for training and 87721 for validation. We rely on the confidence of the classifier to filter out non-existing objects in the image. For testing, we use VilBERT to generate 45000 captions with the visual context using 5K Karpathy test split (Beam search k= 9).

COCO-overlapping. Inspired by that investigated the object count in image captioning, we also created overlapping objects with a caption as a dataset as shown at the bottom of Figure 1. It consists of 71540 overlap annotated captions and visual context.

Despite we extract the top-3 objects from each image, we use three filter approaches to ensure the quality of the dataset (1) Threshold: to filter out predictions where the object classifier is not confident enough, and (2) semantic alignment with semantic similarity to remove duplicated object. (3) semantic relatedness score as soft-label: to grantee the visual context and caption have strong relation, we use Sentence RoBERTa to give a soft label via cosine similarity with a threshold to annotate the final label (if th > 0.2, 0.3, 0.4 then [1,0]). Figure 2 shows the visual context dataset with a different threshold. Note that we omit a higher threshold as the data become imbalanced with a more negative label.

Experiment

Task. To evaluate the dataset, we frame re-ranking task, where the task is re-ranking the caption hypotheses produced by the baseline beam search using only similarity metrics. However, unlike the previous work , we rely only on the similarity to re-rank the caption using semantic similarity \(sim(\text{visual}, \text{caption})\), and thus, we use multi-visual top-3 information at the same time. We use BERT based model to compute the similarity/relatedness as follows:

BERT . BERT achieves remark result in semantic similarity, and thus we fine-tune BERT\(_{\text{base}}\) on the proposed dataset with a binary classifier, cross-entropy loss function [0,1] (related/not related).

BERT-CNN. To take advantage of the overlapping between the visual context and the caption, and to extract global information from each visual, we use the same BERT mentioned above as embedding layer followed by a shallow CNN .

Let \(X=\left\{w_{1}, \ldots, w_{L}\right\}\) be the sentence, where \(w_{i} \in \mathbb{R}^{D}\) is the BERT embedding of the \(i\)-th word in the sentence, and \(L\) denotes the sentence length. We pass the sentence \(X\) through a Kernel \(\mathbf{f} \in \mathbb{R}^{K \times n \times D}\) that convolved over a window of \(n\) words with Kernel size \(K\). By doing this operation, we generate local features of words \(n\)-gram fragments. The local feature of each \(i\)-th fragments is computed:

\[ \small z_{i}=\operatorname{R} \left(\mathbf{f} * w_{i: i+n-1} + b \right) \]

where * is the convolution operator, \(b\) is the bias, and \(\operatorname{R}(\cdot)\) is the Rectified Linear Unit (ReLU) function . By applying this convolution to all the sentence or text fragments, we obtain the corresponding feature map for the n-grams at all locations:

\[ \mathbf{z}=\left[z_{1}, z_{2}, \ldots, z_{L-n+1}\right] \]

where the \( \mathbf{z}\) or \( \mathbf{z}^{(n)} \) is computed using BERT embedding, and each feature map has size (\(n=3)\) . Then, all feature maps are finally concatenated with pooling and then classifier sigmoid layer. We experiment, first, with tuning the last BERT upper 3 layer (BERT-3L) to capture more semantic Note that lower layers captures lexical information and this work is more interested in context information. with CNN. However, we gain more improvement in some metrics when training the 12 layers in an end-to-end fashion.

Evaluation Metric. We use the official COCO offline evaluation suite, producing several widely used caption quality metrics: BLEU METEOR , ROUGE , CIDEr , and BERTscore or (B-S) .

Human Evaluation. We conducted a small-scale human study to investigate human preferences over the visual re-ranked caption. We randomly select 19 test images and give the 12 reliable human subjects the option to choose from two captions: (1) baseline and (2) similarity-based visual ranker. We can observe that in around 50% of the cases the human subject agreed with our re-rank.

Baseline. We use visual semantic information to re-rank candidate captions produced by out-of-the-box state-of-the-art caption generators. We extract top-9 beam search candidate captions from a general pre-trained language and visual model: VilBERT , fine-tuned on a total of 12 different vision and language datasets such as caption image retrieval and visual question answering.

Implementation. We apply different similarity-based re-ranker as shown in Table 1. The re-ranker is a similarity measure score via Fine-tuning BERT between the visual context and the candidate caption. The model is fined-tuned on each dataset that is labeled with a diffident threshold as shown in Table 1, with batch size 16 for 1 epoch with learning rate 2e-5 and max length 50, we kept the rest of hyperparameter settings as the original implementation. For the BERT-CNN the model is trained as end-to-end for 5 epochs.

Result and Discussion

We performed a simple experiment, and we compared the performance of several existing baselines that im-prove caption with object information. All baselines are trained on the same dataset, object-based word re-ranking , an LSTM with object counter , and a language grounding based caption re-ranker . The experiment consists of re-ranking the caption produced by the baseline pre-trained model (language and visual) using only the similarity. In this experiment, each candidate caption is compared to multiple objects and concepts that appear in the image and re-ranked according to the obtained similarity scores. The result of our model and comparison against different baselines are reported in Table 1. The improvement is across all metrics with BERT except BLEU and SPICE. Therefore, we added CNN on the top of BERT to capture word-level glob information and thus we gain improvement over word-level as shown in Figure 3.

Experiments with Pre-trained Model.. Although this approach is proposed to take the advantage of the dataset, we also investigate the use of out-of-the-box based similarity re-ranker on the generated text set with visual context. For this, we use similarity to probability SimProb but we only rely on similarity and the confidence of the classifier as:

\[ \small \operatorname{\text{P}}(w \mid c)=\operatorname{sim}(a, c)^{\operatorname{\text{P}}(c)} \]

where \(\operatorname{sim}(a, c)\) is the similarity between the visual contexts and the caption, and \(\operatorname{\text{P}}(c)\) is the visual classifier averaged confident. We rely on two variations of RoBERTa based model to compute the similarity: (1) SBERT that tuned on sts-b dataset and (2) a contrastive learning based semantic embedding SimSCE (supervise with NLI dataset and unsupervised version). The unsupervised version tries to predict the sentence itself using only dropout as noise. In particular, the model passes the sentence twice with dropout to obtain two embedding as positive pairs then the model predict the positive one among other sentence in the same mini-batch as negative.

Diversity Evaluation. We follow the standard diversity evaluation : (1) Div-1 is the ratio of unique unigram to the number of words in the caption (2) Div-2 the ratio of unique bigram to the number of words in the caption, (3) mBLEU is the BLEU score between the candidate caption against all human captions (lower value indicate diversity) and finally (4) Unique word in the caption before and after re-ranking. Although the two first Divmetric couldn't capture the small changes, our results have lower BLEU and more unique words per caption. Also, we use SBERT (SB) to measure the semantic diversity at sentence level between the desired caution against the 5 human annotations.

Limitation. The drawback of this dataset is that the visual classifier struggle with complex background (i.e., wrong visual, object hallucination as shown in Figure 6. Another limitation is the false positive/negative cosine label score, which leads to the wrong annotation of the relation between the visual and caption. For example, a "paddle" and "a man riding a surfboard on a wave" have low cosine scores. These problems can be tackled by either using Amazon Mechanical Turk or using a more computational expansive visual classifier as Bottom-Up Top-down Feature and Transformer based object detector and semantic segmentation based model.

Conclusion#

We have proposed a simple textual dataset that extends COCO with visual context information, which we believe is useful for the NLP task as current datasets are visual feature-based. This dataset can be used to leverage any text-based task such as learning semantic relation/similarity between visual context and candidate caption, either as post-processing or end-to-end training. Also, we proposed a task and an application that can take advantage of this dataset.
